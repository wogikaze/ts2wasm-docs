# Coverage Matrix

`artifacts/coverage/reference-coverage-matrix.md` is the generated snapshot. This document explains how to read it and when to update it.

## Generator

```bash
python scripts/manager.py update-coverage-matrix
python scripts/manager.py update-coverage-matrix -- --check
```

## Columns

| Column | Meaning |
|---|---|
| `denominator` | total known cases in a reference suite |
| `executed` | cases actually run in the recorded command |
| `build_pass` | cases that compile/build |
| `semantic_pass` | cases that pass semantic checks |
| `executable_build_pass` | build-pass cases that can run |
| `differential_pass` | generated output matched reference output |
| `negative_compile_pass` | negative cases correctly rejected |
| `fail` | executed cases that failed outside unsupported classification |
| `unsupported` | executed cases classified as unsupported |
| `blocked` | harness/toolchain blocked cases |
| `evidence` | command that produced the row |

## Reading rules

- Build coverage is not semantic coverage.
- A sampled row is not a full-suite guarantee.
- Unsupported feature tables are triage inputs, not automatic issue creation.
- Keep the evidence command with the numbers.
